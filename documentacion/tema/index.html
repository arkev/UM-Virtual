Theme Installation

Installing the Theme

To install this theme, you must have WordPress installed on your server. If you need to install WordPress, please follow this link.

Before installing this theme, it is recommended to deactivate all the plugins installed by the previous themes.

You have two options to install the theme:

  1. Using FTP: Unzip the inceptio.zip archive and copy the inceptio folder to wp-content/themes.
  2. Using WordPress: Upload the inceptio.zip archive through WordPress' graphical interface. To do this, go to Appearance - Install Themes - Upload.

After the theme has been uploaded, you need to activate it from Appearance - Themes.

Installing the Child Theme

After installing the Inceptio WP theme, we highly recommend you to install and activate the Inceptio WP Child Theme. Thus, you can custmoize the theme based on your needs without being necessary to modify the source code of Inceptio WP.

The child theme can be found in the archive downloaded from Theme Forest and it is called inceptio-child.zip. The installation steps are similar to the installation steps in the Inceptio theme.

IMPORTANT! Before installing the child theme, YOU MUST upload and activate the Inceptio WordPress theme. After that, you can upload and activate the child theme.

Importing the Demo Content

You have three options to import the demo content.

1. Default WordPress Import

The first option is the one offered by WordPress. This allows you to import the pages, posts, and images, but not the theme settings. To use this method, in Tools - Import - WordPress, upload the demo-content/demo-content.xml file. After the upload has been performed, don't forget to check Download and import file attachments in Import Attachments.

2. Clone Import

The second option is to create a clone of the Inceptio demo database on your environment. This method is developed entirely by Ixtendo and can be accessed from Appearance - Import Demo.

IMPORTANT! Before installing the demo, it is mandatory to install and activate the Revolution Slider WordPress plugin. This plugin can be found in the plugins folder located in the downloaded archive.

The only information that you must provide before starting the clone import is:

  • Site Address (URL) – your WordPress URL address
  • E-mail Address – your email address

We strongly recommend you to RUN THIS IMPORT ON A NEW DATABASE because, after the import is executed, all previous data will be erased definitively and replaced by the demo data.

This plugin has been tested several times on our local environment (WordPress 3.5.2) and it works OK. We cannot guarantee that the import will work without problems in your case. This depends on your environment (database, WordPress version) settings.

PLEASE NOTE THAT WE ARE NOT RESPONSIBLE FOR ANY DATA LOSS.

After the demo import is finished, you must follow these steps:

  • Go to Appearance - Themes and check if the Inceptio theme is activated. If not, then activate it again.
  • Reset the permanent links by following the next steps:
    • Select the Default radio button and click the Save button.
    • Select the Post name radio button again and click the Save button.
  • Copy the 2013 folder (that contains all the photos in the demo) from the demo-images folder located in the downloaded archive to the wp-content/uploads folder in your WordPress installation.

If the clone import is performed successfully, then the following message should appear: "The data import has been successfully performed."

3. Manual SQL Import

The third option is to import the data manually. You can find the SQL script in the demo-data folder.

We strongly recommend you to EXECUTE THIS IMPORT ON A NEW DATABASE because, after the import is executed, all previous data will be erased definitively and replaced by the demo data.

This script has been tested several times on our local environment (MySQL 5.5) and it works OK.

PLEASE NOTE THAT WE ARE NOT RESPONSIBLE FOR ANY DATA LOSS.

To execute this script you must execute the following steps:

  1. Install the Inceptio WP theme and activate it.
  2. Install the Revolution Slider WordPress plugin and activate it. This plugin can be found in the plugins folder located in the downloaded archive.
  3. Open the demo.sql file and:
    1. replace the info@somsite.com with your email address;
    2. replace the http://somesite.com/demo with your WordPress URL address.
  4. Open the MySQL console and execute the following commands:
    1. use inceptio_wp; (replace inceptio_wp with your database name. This command selects the database on which you will execute the import)
    2. source c:/demo.sql; (replace c:/demo.sql with the path to your demo.sql file)

You can take a look at this video to see how I executed this script on my computer: https://www.youtube.com/watch?v=AIhQm9DJhPM.

After the demo import is finished, you must follow these steps:

  • Go to Appearance - Themes and check if the Inceptio theme is activated. If not, then activate it again.
  • Reset the permanent links by following the next steps:
    • Select the Default radio button and click the Save button.
    • Select the Post name radio button again and click the Save button.
  • Copy the 2013 folder (that contains all the photos in the demo) from the demo-images folder located in the downloaded archive to the wp-content/uploads folder in your WordPress installation.

When you are using the shortcodes, you will see that some empty paragraphs are added to the HTML source code after each shortcode. These empty paragraphs are added by WordPress. To avoid them, in the custom.php file or in your Inceptio Child functions.php file (in case you are using the child theme), you must add the following code:

remove_filter( 'the_content', 'wpautop' );
add_filter( 'the_content', 'wpautop' , 99);
add_filter( 'the_content', 'shortcode_unautop', 100);
add_filter('the_content', 'inc_shortcode_format', 101);
if (!function_exists('inc_shortcode_format')) {
    function inc_shortcode_format($content)
    {
        $content = preg_replace('/(<br \/>)\s*(<)/', '<', $content);
        $content = preg_replace('/(<p>)\s*(<div)/', '<div', $content);
        $content = preg_replace('/(<p>)(<a)/', '<a', $content);
        $content = preg_replace('/(<\/div>)\s*(<\/p>)/', '</div>', $content);
        return $content;
    }
}

After you apply the above code, you have to manually insert the paragraphs in shortcodes, like this:

[gc ...]<p>column 1</p>|<p>column 2</p>[/gc]
or 
[gc ...][p]column 1[/p]|[p]column 2[/p][/gc]

Creating a Custom Home Page

By default, the home page is the blog page. If you want to change it with a custom one, you must create a new page using the Full Width template. After that, you will need to go to Settings - Reading and select your page as Front Page Displays.

To configure the slider for the Home Page, click the Edit Slider button and select the slider type and provide the Slider ID.

Creating the Portfolio Overview Page

If you installed the demo content, in the Pages section you will see a page called Portfolio Overview. This page displays all the portfolio projects and has a filtering option. If you haven't installed the demo content, then you must create a new page using the Portfolio Overview template. The page content must be empty. If you want to set this page as a homepage, follow the above instructions.

Creating the Portfolio Page

In the Portfolio section, click add new to create a new portfolio page.

The Portfolio page can have three formats:

  1. Standard – contains one image (use Featured Image for this)
  2. Video – contains a video (Vimeo or Youtube)
  3. Gallery – contains a slider

When you select the format, a button will appear next to the Add Shortcut button which allows you to insert a video or a slider.

Depending on the chosen format, you will be able to insert the following media types in the portfolio:

  1. A single image
  2. Vimeo or Youtube video

To add a video to your portfolio slider, you must select the video format and click the Edit Video button. A pop-up widget with two fields will be displayed (Video ID and Video Content).

In the Project Details section, you have several input fields which allow you to add project details:

  1. Customer – the customer name
  2. Year – the project release year
  3. Technologies Used – the list of technologies used in the project (one by line)
  4. Project URL – the URL to the live project

To provide a short description about the project, you can use the Excerpt section in WordPress. You can find this section above the Project Details section.

Creating the Blog Overview Page

After you install the theme, the home page will be the blog page. If you want to change the home page with another one and to create a blog page separately, you must follow the next steps:

  1. Create a new page in the Pages section. This page will have no content and the Default Template will be set as the page template.
  2. In the Reading Settings section, select your custom page as a Front Page and the newly created page as a Post Page.

Creating the Blog Post Page

In the Posts section, click add new to create a new blog post page.

The Blog Post page can have the following formats:

  1. Standard – contains one or more images
  2. Image – contains an image
  3. Gallery – contains a video
  4. Audio – contains an audio
  5. Video – contains a video
  6. Aside – contains a short piece of text
  7. Quote – contains a quotation
  8. Link – contains an external link in the title

When you select the format, a button will appear next to the Add Shortcut button which allows you to to insert images, audio, video or links.

Depending on the chosen format, you will be able to insert the following media types in the blog post page:

  1. A single image
  2. An image slider
  3. Vimeo or Youtube video

To add a slider to your blog post, you must select the gallery format and click the Edit Slider button.

A dialog will be open and, here, you can configure the slider.

Creating the Contact Page

If you installed the demo content, the contact page is already created for you. By default, the messages are sent to your email address which must be specified in Settings - General - E-mail Address.

If you didn't install the demo content, then, to create the contact page, you must follow the next steps:

  1. Create a new page and name it Contact.
  2. Choose the Contact Page - Right Sidebar or Contact Page - Without Sidebar as a template.
  3. Open the Contact Editor and configure the address and the Google Map coordinates.

If you leave the page content empty, then the default contact form will be displayed.

Theme Options

You can customize this theme in Appearance - Theme Options. Here are six categories that contain the following options:

  1. General Section
    • SEO Settings – define the global description and keywords that will be included in all pages. These keywords are used by the search engines. If you want to have different keywords for the various pages, you can do this by going to the edit page mode for that page and setting the new keywords in the Keywords, Description, etc., input fields in the SEO Settings section. This section can be found at the bottom of the page editor.
    • Header Settings – configure the page header.
    • Footer Settings – configure the page footer.
    • Footer Featured Settings – configure the page footer featured section (will appear above the footer).
    • MailChimp Settings – MailChimp configuration (this must be done in case you want to use the newsletter subscription widget).
    • ReCaptcha Settings – ReCaptcha configuration.
    • Contact Details – configure the contact details: address, email, phone, fax, etc. This information will be used by the contact widgets.
    • RSS Settings – enable the RSS for this website. You can choose the RSS version, too. The RSS button will be displayed at the bottom of the page (at the end to the social icons). By default, the RSS 2.0 feed is enabled.
    • Enable Responsiveness – you can disable or enable the HTML5 responsivness.
  2. Blog & Portfolio
    • Blog Settings – allows you to configure the blog post pages.
    • Portfolio Settings – allows you to configure the portfolio pages.
  3. Social – This section contains the settings for the most common social networks. This information is used by the custom widgets.
  4. Layout – This section allows you to choose between the Wide and Boxed layout.
  5. Styles & Typography (Advanced) – This section contains the CSS properties that allow you to change the background patterns, the font colors, style, size, etc.

After you create a custom form, you can process its submission using the same function which is used to process the default contact form. To do this, you must set the value of process_contact_formas action.

If you want to process your custom form submission differently, then you must create a custom PHP function in:

  1. functions.php file located in the Inceptio Child theme (in case of using the child theme). This method is highly recommended.
  2. custom.php file located in the Inceptio theme. In this case you must backup this file each time when you update the theme.

The name of the function must respect the following pattern: inceptio_[action name] where the [action name] is the value of the action attribute in the [form] shortcode.

For example, if you create a form using the value of opinions as an action, then the function name must be inceptio_opinions.

Upgrading the Theme

We strongly recommend you to backup your current theme version before upgrading it.

You can apply this method safely only if you didn't change the theme's source code (the CSS files or the PHP files).

To upgrade the theme using this method, you must follow the next steps:

  1. Download the latest archive with the theme from your Theme Forest account.
  2. Unzip the downloaded archive on your computer. Inside, you will find a folder called inceptio. This folder contains the WordPress theme.
  3. Override the custom.css and custom.php files in this folder with the content of these two files from your old theme (to preserve the customizations).
  4. Override all the translation files located in the lang folder of the new theme with the translation files located in the lang folder of your old theme (current theme), to preserve the translation.
  5. Open each .po file in the lang folder using PoEdit, click update button and save the document (only if the second step was performed).
  6. Archive all the content of the inceptio folder. This will be the zip archive with the new version of Inceptio.
  7. Delete the current version of your Inceptio WordPress theme. (Don't worry. When you delete a theme from WordPress, only the PHP files, CSS files and the theme images will be deleted. The other information will remain untouched: uploaded pictures, posts, pages, portfolio, theme options, etc.)
  8. Upload the zip archive created at step no. 6 and activate the theme.
